Scheduled ticks

Scheduled ticks, also known as tile ticks, are the method by which Minecraft remembers to do something in the future. They happen in their own tick phase. Here's how they work:

there is a "world time" attribute, which indicates the number of ticks this Minecraft world has been running. When a tick is scheduled, the game stores the following data:

  • The time when the tick should be executed
  • The location of the scheduled tick
  • The block that scheduled the tick

When it comes time for a tick to be executed, the game first checks that the block in the location is of the correct type. This is so that, if you schedule a repeater tick and then use a piston to push an observer into place before it can fire, the observer doesn't take up the repeater's scheduled tick and fire early.

Then, it calls the block's scheduled tick function. This can do many different things; here's a non-comprehensive list, with the delay of the scheduled tick:

  • Observers toggle their state (2gt)
  • Torches, repeaters, and comparators check for power (2/4/6/8gt for repeaters, 2gt otherwise)
  • Gravity blocks fall (1gt)
  • Scaffolding and leaves change their distance attribute (1gt)
  • Composters create bone meal (You know that little delay between the last layer of compost and then bone meal actually being formed? Yeah.) (20gt)

Scheduled ticks, generally, happen in the order that they were scheduled. However, this is not always true in the case of repeaters and comparators. See the tile tick priority page for more details.
